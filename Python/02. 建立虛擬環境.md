在 Python 中，python -m venv venv 是最標準且內建的虛擬環境建立方式。這裡的兩個 venv 意義不同：第一個 venv 是 Python 的模組名稱，第二個 venv 是你要建立的資料夾名稱（你也可以改叫 .env 或 myenv）。

以下是完整的操作流程與原理：
1. 建立虛擬環境的步驟

請在 VS Code 的終端機（Terminal）執行以下指令：
Step 1: 建立環境
Bash

python -m venv venv

執行後，你會發現專案目錄下多了一個名為 venv 的資料夾，裡面包含了獨立的 Python 執行檔與套件存放區。
Step 2: 啟動環境 (關鍵步驟)

建立完後必須「進入」該環境，否則安裝的套件仍會跑到系統全域。

    Windows:
    PowerShell

    .\venv\Scripts\activate

    macOS / Linux:
    Bash

    source venv/bin/activate

    提示： 啟動成功後，終端機提示字元的前面會出現 (venv) 的字樣。

Step 3: 確認環境

輸入以下指令確保路徑指向專案資料夾內的 Python：
Bash

which python    # macOS/Linux
where python    # Windows

2. 在 VS Code 中整合虛擬環境

在終端機啟動後，VS Code 有時不會自動切換編輯器的解譯器。為了讓代碼補全和語法檢查正常工作，請務必執行：

    按下 Ctrl + Shift + P (Mac 是 Cmd + Shift + P)。

    搜尋並選擇 Python: Select Interpreter。

    選擇標記為 'venv': ./venv/Scripts/python.exe 的選項。

3. 為什麼要這樣做？ (核心原理)

如果不使用虛擬環境，所有的 AI 套件（如版本要求嚴苛的 PyTorch 或 TensorFlow）都會裝在一起。

    防止版本衝突： A 專案要用 NumPy 1.21，B 專案要用 NumPy 2.0，虛擬環境能讓它們互不干擾。

    乾淨的部署： 當你執行 pip freeze > requirements.txt 時，它只會記錄「這個專案」需要的套件，而不會把你電腦裡幾百個無關的套件都抓進去。

4. 常用管理指令
<pre>
動作	指令
安裝套件	pip install <package_name>
查看已安裝套件	pip list
導出環境清單	pip freeze > requirements.txt
退出虛擬環境	deactivate
刪除虛擬環境	直接刪除 venv 資料夾即可
</pre>

常見陷阱： 在 Windows 上執行 activate 時，若出現「因為此系統上禁止執行指令碼」的錯誤，這是因為 PowerShell 的權限限制。您可以改用 Command Prompt (cmd)，或者在 PowerShell 執行 Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope Process 來暫時開啟權限。
