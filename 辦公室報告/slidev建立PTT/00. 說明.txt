專案路徑設定錯誤通常是因為在執行 npm init slidev 時，路徑中包含了中文、空格，或是權限不足的資料夾（如 C 槽根目錄）。

別擔心，我們不需要去修補那個壞掉的資料夾，最快的方式是「徹底清除並換個地方重新開始」。請按照以下步驟操作：
1. 強制刪除失敗的殘骸

在終端機（CMD 或 VS Code 終端機）中，確保你不在那個報錯的資料夾內，然後執行：

    Windows (PowerShell): Remove-Item -Recurse -Force 你的資料夾名稱

    Mac/Linux: rm -rf 你的資料夾名稱

    注意： 如果刪不掉，請關閉所有開啟該資料夾的 VS Code 視窗或檔案總管，再試一次。

2. 選擇「安全」的路徑重新開始

Slidev 對路徑比較挑剔，請遵循以下原則選擇新位置：

    不要有中文： 例如 C:\桌面\我的簡報 (❌) → C:\Projects\my-slide (✅)

    不要有空格： 例如 C:\My Documents (❌)

    路徑不要太長： 建議直接在 D 槽或使用者目錄下建立一個 Projects 資料夾。

3. 正確的初始化指令流程

請打開終端機，依序輸入以下指令：
Bash

# 1. 切換到一個乾淨的磁碟路徑 (例如 D 槽)
d:
cd \

# 2. 建立一個專屬資料夾並進入
mkdir slide-project
cd slide-project

# 3. 執行初始化 (建議加上名稱防止路徑混亂)
npm init slidev@latest .

(注意最後那個 . 代表「就在當前資料夾初始化」，這能避免路徑再次跑偏。)
4. 解決「指令抓不到」的問題 (常見錯誤)

如果初始化後執行 npm run dev 出現「找不到指令」，請檢查：

    是否安裝了依賴： 在資料夾內執行 npm install。

    Node 版本： 輸入 node -v。Slidev 2026 年版本通常需要 Node 18.0 以上。

5. 萬用備案：如果電腦環境一直搞不定

如果你發現本機路徑問題層出不窮，推薦你使用 線上開發環境，完全跳過路徑與安裝問題：

    StackBlitz: 瀏覽器打開 sli.dev/new，它會直接幫你開好一個完全設定好的 Slidev 環境，你只要改左邊的 slides.md 即可。
