萬用結構化 Prompt 範本
# Role (角色)：你是一位擁有 10 年經驗的 [請填入職位，如：SEO 專家/金牌業務]。
# Task (任務)：我的目標是 [請填入具體目標，如：寫一篇關於 AI 的部落格大綱]。
# Context (背景)：這份內容是給 [請填入受眾，如：完全不懂科技的小白] 看的。目前的痛點是 [請填入細節，如：他們覺得科技很冰冷]。
# Format (格式)：請使用 [請填入格式，如：Markdown 格式/心智圖結構] 呈現，並包含 [請填入必要項目，如：前言、三個重點、結論]。
# Tone (語氣)：語氣請保持 [請填入語氣，如：親切、有活力且平易近新]。
# Example (範例)：請參考以下風格：[可貼上你喜歡的一段文字或數據範例]。


針對程式設計師 (Programmer) 的情境，最常見的需求不外乎是：寫新功能、優化效能、重構程式碼或是Debug。
這裡為你準備一個「高階工程師代碼生成/重構」的專屬範本，這套 Prompt 能確保 AI 不只是「寫出能跑的程式」，而是產出「高品質、易維護」的代碼。
💻 程式設計師專用：結構化 Prompt 範本
# Role (角色)
你是一位資深的 [填入語言，如：React/Python/Go] 全端架構師，擁有超過 10 年的系統開發與大型架構設計經驗，極度重視 Clean Code、設計模式與程式碼效能。
# Task (任務)
請幫我 [填入具體任務，如：實作一個處理大型 CSV 的異步上傳功能 / 重構現有的 API 邏輯]。
# Context (背景)
環境：使用 [填入框架與版本，如：Node.js v18 / Django 4.2]。
限制：需要考慮 [填入限制，如：記憶體佔用、API 延遲、安全性]。
邏輯說明：這段功能主要是為了 [描述你的需求邏輯]。
# Example (範例/參考)
請遵循 [如：SOLID 原則 / Airbnb Style Guide]。
變數命名請使用 [如：camelCase / snake_case]。
# Format (格式)
代碼區塊：完整的程式碼實作。
邏輯解說：用條列式說明關鍵技術決策（為什麼這樣寫）。
潛在風險：列出未來可能需要注意的邊界情況 (Edge Cases)。
單元測試：提供一段簡單的測試範例。
# Tone (語氣)
專業、技術導向、簡潔明瞭，不需要過多的寒暄。

實戰小技巧 (Pro Tips)
身為開發者，在使用 AI 時建議加入這兩行「隱藏咒語」，效果會更好：
「請在程式碼中加入必要的註解，解釋複雜邏輯。」
「如果你認為有更好的實作方式（如更好的演算法或現成的函式庫），請直接提出建議並說明原因。」
你想試試看針對某個具體的「功能需求」或「Bug」來測試這個範本嗎？ 如果你有現成的程式碼片段要我幫你優化，也可以直接貼上來！
