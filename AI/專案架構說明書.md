## 3. 核心邏輯流 (Core Data Flow)
1. **Request**: 使用者透過 UI 或 API 端點發起請求。
2. **Business Logic**: 邏輯判斷由 `services/` 進行處理。
3. **Data Access**: 透過 Data Client 與資料庫或外部 API 通訊。
4. **Response**: 統一格式化後回傳至前端或調用者。

## 4. 開發規範與慣例 (Conventions)
*   **命名規範**: 檔案使用 `kebab-case`，元件使用 `PascalCase`。
*   **型別要求**: 嚴禁使用 `any`，所有 API 響應必須定義 Interface。
*   **錯誤處理**: 全域錯誤捕捉器位於 `core/error-handler`。

## 5. 模組相依性規則 (Dependency Rules)
*   `api` 層僅能調用 `services`，不可直接操作資料庫。
*   `services` 負責業務檢核，不得直接處理 HTTP 狀態碼。
*   `utils` 必須是純函數，嚴禁引用業務邏輯。
